《Web应用基础结业报告》

一、做的什么

  • 这是一个用Angular前端开发框架和Angular Material组件库做的一个收集狗狗信息的动态网页。
  • 主页是由一个选项卡和一个搜索框组成，选项卡初步展示了所有狗狗的信息，使用搜索框搜索狗狗名称能看到相应狗的图片等详细信息。
  • 管理页面由一个展示所有狗的列表、三个输入框和一个按钮组成，点击列表中相应选项能查看或删除相应狗的信息，在输出框中输入相应信息并点击Add Dog按钮能新增狗狗信息，新增的信息能同步在主页和列表显示。
  • 狗狗的详细信息页面由一个Angular Material卡片、三个输入框和两个按钮组成，卡片展示狗狗的图片和详情，在输入框中修改狗狗信息后点击Save按钮能修改狗狗的信息，点击Go Back按钮能返回操作的上一个页面。

二、开发过程

  • 本网站基于英雄之旅开发，在英雄之旅的基础上做了一些功能的延伸和样式的设计。

（1）方便起见，首先按照英雄之旅教程，创建基础的hero类和dashboard、search、service、路由等组件，也就是以英雄之旅教程那个简单的网页作为模板。

（2）用ng add @angular/material命令添加Angular Material组件库，然后导入自动完成、按钮、图标、卡片等需要用到的组件模块，至此前期准备工作就做好了。

（3）思考之后，决定做一个狗狗信息的收集网站，然后到网上收集了不同狗狗的描述、图片等资料。

（4）对基础的对象hero的类进行修改，除了id和name属性以外，添加了describe、img属性，分别表示狗狗的描述和图片。

（5）先对网站的样式进行简单的设计，将<input>、<button>等标签的样式替换为Angular Material的样式，使其看起来更美观，网页的背景也添加上去。dashboard页面一开始的4个Top Heroes不是很符合审美要求，显得比较突兀，将其去掉替换为Angular Material的选项卡，显示的内容与hero关联，选项卡名称为狗狗名称，显示的信息为狗狗的描述。heroes页面的布局也不是那么美观，将列表的<a>标签替换为透明的<button>,x按钮修改为mat-icon的close图标，看上去更和谐，再讲包裹id的<span>标签的颜色替换为橘红色，与网站的主题色搭配。列表右边的区域空了一大块，正好将添加操作移动过去，并设置三个输入框，分别对应名称、描述、图片三个属性，id是自动添加的。details详情展示页面太过单调，使用Angular Material的卡片来展示狗狗详细信息更加美观，将信息修改操作设置在卡片的右侧，三个输入框也分别对应除id外的三个属性。

（6）完善相应操作的ts代码，写好Add、Delete、Update、Search函数，与相应的按钮绑定，并通过subscribe()等方法传递数据，使各个功能能正常工作。

（7）注入title组件，重新设置网页的标题，改为Dogs Collection。

（8）最后再对页面细节进行美化，例如列表元素之间添加分割器等，使页面看起来更美观

三、遇到的问题

（1）安装内存 Web API 包后，突然报错找不到tslib。

（2）使用Angular Material的input组件时，搜索框名称与搜索框的下划线相隔距离较远，并且搜索匹配结果列表会显示在下划线上方。

（3）使用卡片组件时，使用本地图片无法显示。

（4）使用选项卡组件时，与复写器指令*ngFor结合使用时出现了多个选项卡，而不是预想中的一个选项卡多个选项。

（5）背景图片的全屏有点问题，如果里面的元素超出了屏幕，往下滑动的时候就不是全屏了。

四、如何解决

（1）执行npm update命令后解决。

（2）将搜索匹配结果列表的<mat-option>标签添加在自动完成的<mat-autocomplete>标签中

（3）最开始是将图片放在新浪微博上，本地运行图片可以显示，部署在GitHub上时却无法访问，最后使用百度图片的源地址才解决。

（4）将复写器指令*ngFor的<div>放在选项卡的<mat-tab-group>标签之间，包裹<mat-tab>属性。

（5）不能将height属性设置为固定的100%，应该设置最小高度min-height的值为100%。

五、哪些未解决

（1）网页的标题左侧的图标没弄清楚怎么修改。

（2）网页无法根据屏幕大小动态调整适合的位置，卡片组件自身倒是能自动调整，但网页整体无法实现。
